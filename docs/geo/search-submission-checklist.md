# Search Submission Checklist

This checklist covers the few steps that usually still require manual action outside the repo.

## Already Done In The Repo

- landing page for the public site
- `robots.txt`
- `sitemap.xml`
- `llms.txt`
- clearer page titles and descriptions

## Manual Steps

### 1. Google Search Console

- add and verify the site
- submit the sitemap:
  - `https://lujuncheng1225-cloud.github.io/AI_Commercialization--Product-Management-skills/sitemap.xml`

### 2. Bing Webmaster Tools

- add and verify the site
- submit the same sitemap

### 3. Monitor Indexing

Check whether these pages are being discovered:

- homepage
- manifesto
- case studies
- method framework
- signature cases

### 4. Publish Supporting Content

Search visibility improves faster when the repo is linked from:

- social posts
- personal profiles
- blog posts
- community shares

## Recommended First URLs To Share

- `/`
- `/pm-skills-interactive-course.html`
- `/core/MANIFESTO.html`
- `/brand/WHY_THIS_REPO.html`

## One-Line Advice

**Search discoverability is not just a repo problem; after the technical basics are in place, indexing and external links are the next meaningful steps.**
