## 推荐流程

### 1. 打开 Search Console

进入：

- <https://search.google.com/search-console>

### 2. 添加资源

推荐直接添加：

- `URL prefix`

填写：

- `https://lujuncheng1225-cloud.github.io/AI_Commercialization--Product-Management-skills/`

为什么建议用这个：

- 对 GitHub Pages 更直接
- 不需要你控制根域名
- 验证路径更清晰

### 3. 完成验证

如果 Google 提供多个验证方式，优先选择它当前最容易完成的方式。

通常对 GitHub Pages 来说，常见做法是：

- HTML 文件验证
- HTML meta tag 验证

如果它要求你加验证文件或 meta tag，再回来，我可以继续帮你把对应文件放进仓库。

## 你做完后应该看到什么

理想状态下，你会逐步看到：

- 资源验证通过
- sitemap 状态正常
- 部分页面变成 `已发现`
- 后续逐步出现 `已编入索引`

## 重要预期

提交并不等于立刻有流量。

它解决的是：

- 让 Google 更快发现页面
- 让页面更早进入索引流程

真正的变化通常还是需要：

- 几天到几周
- 外部链接
- 持续更新
