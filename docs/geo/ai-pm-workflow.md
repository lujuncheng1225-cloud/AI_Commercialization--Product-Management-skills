# AI PM Workflow For PRDs And Feature Shaping

A strong AI PM workflow should not start with generation.

It should move through five steps:

1. clarify the problem
2. decompose the work
3. make the core decision
4. document the result
5. review before execution

## Why This Workflow Matters

Many AI PM workflows fail because they jump directly to output.

That creates common problems:

- PRDs written before the problem is clear
- AI features scoped before evaluation is defined
- plans that sound polished but are not operational

## The Recommended Sequence

### 1. Clarify

Ask only the minimum questions needed to improve the conclusion.

### 2. Decompose

Split the task into problem definition, evaluation, risk review, and output creation.

### 3. Decide

Make the main judgment explicit:

- what should happen
- why this path is stronger
- what assumptions still exist

### 4. Document

Turn the decision into an artifact such as:

- a PRD
- an AI feature brief
- an evaluation plan
- a launch review

### 5. Review

Check for:

- weak assumptions
- missing evidence
- conflicting recommendations
- unclear next steps
