# 发布文案

## 长文版

这段时间我把一个原本偏内部使用的 PM skill 仓库，逐步做成了一套更完整的 `PM AI Skill Toolkit`。

它不是一个“提示词大全”，而是一套给产品经理和 AI agent 共同使用的 PM 操作系统。

我想解决的问题很简单：

现在很多 AI for PM 的内容，都在帮大家“更快生成一份东西”。
但真正难的，往往不是写出来，而是：

- 有没有先把问题问对
- 有没有把判断过程结构化
- 有没有把证据、假设和建议分开
- 有没有把一类工作沉淀成可复用流程

所以这套工具现在做的事情不是“多给几个 Prompt”，而是把 PM 工作方式拆成几层：

- 给人直接上手的交互式工具箱
- 可复用的 skills 和 commands
- 面向 agent 的 routing / output standards / evals
- 可逐步补齐的私有上下文模板

目前仓库里已经有：

- 22 个 PM skills
- 10 条多技能工作流
- 交互式 PM 工具箱
- 中英文双语入口
- 商业化、增长、AI 功能定义等核心场景
- 给 Codex / Claude Code / Cursor 的接入方式
- 评估用例和输出标准

它现在更像一套 `AI-native PM operating system`，而不只是一个知识库。

如果你也在思考下面这些问题，这个项目可能对你有帮助：

- AI 时代产品经理该怎么和 agent 协同工作
- 怎么把 PM 工作流做成可复用系统
- 怎么把商业化与增长判断做得更结构化
- 怎么避免 PM 退化成 prompt operator
