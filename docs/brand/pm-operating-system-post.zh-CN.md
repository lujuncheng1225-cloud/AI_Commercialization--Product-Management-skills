# 我不想再把它叫成一个 prompt 库了

这套项目如果还被叫成“PM prompt 库”，其实是被说小了。

我更想用一个词：

`PM Operating System`

因为它真正包含的，不只是 prompt。

而是：

- skills
- commands
- routing
- output standards
- evals
- private context
- benchmark

也就是说，它想解决的不是：

“怎么多写几个场景”

而是：

“怎么让产品经理和 agent 在复杂任务里，按更稳定的方式工作”

这才是我觉得真正值钱的部分。

如果一个系统只能帮你更快写出东西，
那它很容易被替代。

如果一个系统能把：

- 问题定义
- 路由判断
- 证据和假设分离
- 风险与下一步

都做成长期可复用的结构，
那它才更像能力底座。

所以我现在会这样介绍它：

`PM AI Skill Toolkit is a PM Operating System for product managers and AI agents.`

产品名是 `PM AI Skill Toolkit`。
类目词是 `PM Operating System`。

这不是文案游戏。

这是我想把它往哪条路上推的选择。
