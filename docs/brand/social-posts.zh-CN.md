# 社媒短帖

## 用法

这 3 条短帖不是“更新日志”，而是传播入口。

它们的目标是：

- 先抓住注意力
- 再给一个强观点
- 最后把人导回仓库

## 短帖 1：观点型

很多 AI for PM 的内容，重点都是“更快写 PRD”。

但我越来越觉得，AI 对产品经理真正重要的，不是写得更快，而是：

- 问题有没有问对
- 判断有没有收敛
- 证据和假设有没有分开
- 工作流能不能长期复用

所以我做了 `PM AI Skill Toolkit`。
它不是提示词库，而是一套给产品经理和 AI agent 共用的 `PM Operating System`。

- 仓库：[PM AI Skill Toolkit](https://github.com/lujuncheng1225-cloud/AI_Commercialization--Product-Management-skills)
- 体验：[Interactive Toolkit](https://lujuncheng1225-cloud.github.io/AI_Commercialization--Product-Management-skills/pm-skills-interactive-course.html)

## 短帖 2：结果型

我把一个原本偏内部使用的 PM skill repo，做成了一个更完整的 `PM Operating System` 产品：`PM AI Skill Toolkit`。

现在它已经有：

- 交互式工具箱
- 22 个 PM skills
- 10 条多阶段 workflow
- agent routing / output standards / evals
- 商业化和 AI PM 场景

我想做的不是“再加几个 Prompt”，而是把 PM 工作方式沉淀成系统。

- 仓库：[PM AI Skill Toolkit](https://github.com/lujuncheng1225-cloud/AI_Commercialization--Product-Management-skills)

## 短帖 3：方法型

我现在越来越相信，AI 时代产品经理最值得沉淀的，不是模板，而是判断链路：

`澄清 -> 拆解 -> 判断 -> 成文 -> 审查`

如果一个系统不能帮你把这条链路做稳，那它大概率只是让你“写得更像样”，而不是“做得更专业”。

这也是我为什么没有做一个提示词库，而是做了一个 `PM Operating System`：`PM AI Skill Toolkit`。
