# Launch Post

## Long Version

Over the last stretch, I turned what started as an internal PM skills repository into a more complete `PM AI Skill Toolkit`.

It is not a prompt dump.
It is a PM operating system for both product managers and AI agents.

The problem I wanted to solve is simple:

most AI-for-PM content helps people generate artifacts faster.
But the harder part of PM work is usually not writing faster. It is:

- framing the right problem
- structuring judgment
- separating evidence from assumption
- turning repeated work into reusable workflows

So this project is built as a system instead of a prompt library.

It now includes:

- an interactive PM toolkit
- reusable `skills` and `commands`
- agent routing and output standards
- eval cases and rubrics
- optional private context templates

Current scope:

- 22 PM skills
- 10 multi-skill workflows
- bilingual entry points
- AI PM and commercialization PM use cases
- adapters for Codex, Claude Code, and Cursor
- an interactive toolkit you can try in the browser

The direction is straightforward:

help PMs work with AI in a way that improves judgment, not just output speed.

If you care about questions like these, the repo might be useful:

- how PMs should collaborate with AI agents
- how to turn PM practice into reusable systems
- how to add structure to commercialization and growth judgment
- how to avoid turning PMs into prompt operators

Interactive toolkit:
