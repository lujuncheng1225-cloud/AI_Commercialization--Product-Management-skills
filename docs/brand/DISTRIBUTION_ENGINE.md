# Distribution Engine

This file defines the ongoing distribution machine for PM AI Skill Toolkit.

Global reach will not come from repo cleanliness alone.
It comes from repeated, high-signal distribution loops.

## Weekly Flywheel

Every week, publish:

1. one real case breakdown
2. one benchmark comparison or failure pattern
3. one toolkit scenario demo

These three outputs cover:

- proof of value
- proof of rigor
- proof of usability

## Monthly Cadence

Every month, publish:

1. one benchmark update
2. one version update
3. one category-language reinforcement post
4. one directory submission or refresh sweep

## Core Content Units

### Real Case Breakdown

Show:

- the user problem
- the chosen route
- the output structure
- what changed because of the workflow

### Benchmark Comparison

Show:

- task
- expected route
- actual route
- key failure pattern
- what the benchmark teaches

### Toolkit Scenario Demo

Show:

- the scenario card
- the generated brief
- the mapped source `skill` or `command`
- how it escalates from “web use” to “system use”

## Distribution Targets

Maintain a repeatable submission and refresh list:

- awesome lists
- agent skill directories
- AI tool directories
- GitHub topics
- launch communities
- PM communities

The rule:

do not wait to be discovered.
submit, refresh, and re-explain.

## What To Measure

- stars and forks
- profile clicks
- toolkit visits
- benchmark page visits
- copy actions from toolkit into agents
- inbound references from directories, lists, and social posts

## Content Positioning Rule

Do not publish generic “AI for PM tips”.

Publish things that reinforce the category:

- PM Operating System
- Agent-native PM Workflow
- Commercialization Review Framework
- Pricing Packaging Brief
- PM Agent Benchmark

## Owner Checklist

Before each week ends:

- did we ship one case?
- did we ship one benchmark artifact?
- did we ship one toolkit demo?
- did we push one directory or discovery action?

If not, distribution has stalled.
