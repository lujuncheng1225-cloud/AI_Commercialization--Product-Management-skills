# I do not want to call this a prompt library anymore

Calling this project a “PM prompt library” makes it smaller than it really is.

The better term is:

`PM Operating System`

Because the real stack is not just prompts.

It is:

- skills
- commands
- routing
- output standards
- evals
- private context
- benchmark

So the problem it tries to solve is not:

“How do we add more scenarios?”

It is:

“How do we help product managers and agents work more reliably on complex tasks?”

That is the part worth building.

If a system only helps people write faster, it becomes replaceable very quickly.

If a system helps turn:

- problem definition
- routing judgment
- evidence discipline
- risk and next-step structure

into a reusable layer, then it becomes closer to infrastructure.

That is why I now describe it this way:

`PM AI Skill Toolkit is a PM Operating System for product managers and AI agents.`

`PM AI Skill Toolkit` is the product name.
`PM Operating System` is the category term.
