# Start Here

中文入口文档请从这里开始。

如果你只记住三件事：

1. 先选一个 `command`，单点任务再选 `skill`
2. 提一个真实业务问题，不要只给抽象方向
3. 默认按 `澄清 -> 拆解 -> 产出 -> 审查` 的顺序推进

## 默认工作流

如果你不知道从哪开始，按这条顺序最稳：

1. `workshop-facilitation`
   先把需求和边界聊透
2. `task-decomposition`
   把大任务拆成可检查的小步骤
3. 目标 `skill` 或 `command`
   真正产出文档、判断或方案
4. `review-gates`
   交付前跑一轮结构化审查

这套流程尤其适合：

- 写 PRD
- 定义 AI 功能
- 做商业化诊断
- 做复杂评审
