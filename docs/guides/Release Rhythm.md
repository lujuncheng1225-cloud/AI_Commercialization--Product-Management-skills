# Release Rhythm

Use this repository like a product and a benchmark, not just a content library.

## Weekly

- Ship one real case breakdown
- Ship one benchmark comparison, failed route recap, or rubric refinement
- Ship one toolkit scenario demo
- Review new skill requests and bug reports
- Regenerate catalog if new skills or commands were added

## Monthly

- Publish one benchmark update
- Publish one version update or changelog summary
- Publish one category-language reinforcement post
- Refresh at least one directory, awesome list, or discovery surface

## Before a tagged release

1. Update `docs/updates/CHANGELOG.md`
2. Regenerate catalog
3. Review new or edited metadata for trigger clarity
4. Run validation and style consistency checks
5. Spot-check at least one realistic prompt per new skill
6. Confirm toolkit entry, benchmark layer, and docs index stay aligned

## Suggested versioning

- `0.x` while the library is still evolving quickly
- minor release for new skills, commands, or public benchmark structure
- patch release for fixes, wording improvements, or entry-layer polishing
