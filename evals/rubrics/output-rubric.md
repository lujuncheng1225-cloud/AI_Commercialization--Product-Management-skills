# Output Rubric

## Purpose

评估输出是否达到本仓库定义的专业标准。

总分 `7` 分。

## Scoring Dimensions

### 1. Decision Quality (0-2)

- `2`：有明确主结论，且和任务阶段匹配
- `1`：有结论，但模糊、分散或阶段错位
- `0`：没有主结论，或结论明显错误

### 2. Evidence Quality (0-2)

- `2`：给出关键依据，并区分事实、推断和待验证项
- `1`：有依据，但混杂或不够关键
- `0`：基本没有依据，主要是空泛判断

### 3. Professional Structure (0-2)

- `2`：结构符合当前 `command` / `skill`，信息完整，能直接用
- `1`：大体可用，但缺关键块，例如风险、范围边界、上线条件
- `0`：结构明显跑偏，难以复用

### 4. Reliability (0-1)

- `1`：没有臆造数据、背景或不当确定性
- `0`：出现臆造、错误自信或明显空话

## Automatic Failure Conditions / 直接判低分条件

出现以下任意情况，`Output` 不应高于 `3` 分：

- 臆造 KPI、baseline、用户分群、评估结果
- 把 launch review 写成 PRD
- 把优先级推荐写成泛泛方法介绍
- 没有主要风险或下一步动作
- 没有说明关键信息缺失却给出高置信度结论
