# Decision Preferences Starter

> Status: starter draft, not factual personal preference. Edit this until it matches how you actually make decisions.

## General Decision Style

- Prefer fast directional decisions when uncertainty is high and reversibility is good
- Prefer higher-confidence decisions when rollout risk, commercial impact, or trust risk is high
- Prefer experiment-first instead of PRD-first when the main unknown is user value or behavioral response

## Product Decisions

- A full PRD is usually warranted when scope crosses multiple teams, changes a core workflow, or affects rollout / guardrails
- A brief or memo is usually enough for small experiments, problem framing, or early recommendation work
- Escalate when the task is actually strategic alignment, not backlog-level execution

## AI Decisions

- Unsupported claims and silent hallucinations are unacceptable in user-facing flows
- Fallback is mandatory when output confidence is variable and the user may trust the result too much
- Demo quality is never enough for launch; evaluation, monitoring, and recovery path are required

## Growth / Commercial Decisions

- Treat a problem as funnel-first when the main loss is at a specific stage transition
- Treat a problem as pricing / packaging-first only when value moment and segment logic are already reasonably clear
- Require an experiment before a large strategy change when the main uncertainty is user willingness to pay or conversion behavior

## Communication Preferences

- Preferred answer style: concise, structured, decision-first
- Preferred amount of detail: enough to act, not inflated for show
- Preferred structure: decision, evidence, assumptions, risks, next steps

## Notes For Agent

- What kind of answers you trust more: bounded recommendations with explicit tradeoffs and clear missing information
- What kind of answers you distrust: vague strategy talk, fake precision, and confident claims without evidence
