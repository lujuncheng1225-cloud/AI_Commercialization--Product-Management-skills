# Metrics Context Starter

> Status: starter draft, not factual business truth. Replace definitions and thresholds with your real metric system.

## North Star

- Metric name: successful retained workflow completions
- Why it matters: it links user value to repeat behavior better than raw signups or feature clicks
- Current definition: repeated completion of a core task by users or accounts that continue to come back

## Core Growth Metrics

- Activation: first meaningful completion of the core workflow
- Conversion: movement from free / trial / internal pilot to paid or committed usage
- Retention: repeated return to complete the core workflow again
- Revenue: upgrade conversion, revenue per active account, or expansion from higher-value usage

## AI Quality Metrics

- Quality metric 1: task success rate
- Quality metric 2: unsupported claims or error rate
- Quality metric 3: edit rate / override rate

## Guardrails

- Guardrail 1: complaint or refund rate
- Guardrail 2: latency or failed-response rate
- Guardrail 3: fallback trigger rate or manual correction rate

## Definitions

- How do you define activation? User completes the first meaningful task, not just signs up or clicks around.
- How do you define paid conversion? User or account crosses a commercial commitment threshold, not just sees the pricing page.
- How do you define retained user? User returns in a relevant time window and repeats the core workflow.
- How do you define successful AI task completion? User gets an output that is usable with acceptable correction cost.

## Metric Interpretation Rules

- Which metrics are leading indicators? activation quality, time to first value, first repeat usage, edit rate
- Which metrics matter most in decision-making? the metrics closest to durable user value and commercial outcomes
- Which metrics are often misread in your business? raw traffic, raw model output volume, and shallow engagement without repeat value

## Notes For Agent

- Metrics the agent may reference generically: activation rate, trial-to-paid conversion, repeat usage, task success rate, fallback rate
- Metrics the agent must not invent: actual baseline values, targets, deltas, or thresholds unless the user explicitly provides them
