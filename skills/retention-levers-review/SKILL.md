## Purpose / 用途

从重复价值、使用频率和工作流嵌入程度去分析留存。

## Key Concepts / 核心概念

留存往往取决于：

- repeat value 的速度
- 协作或网络效应
- habit loop 强度
- lifecycle 触达设计

## Application / 用法

```markdown
## Retention Review

### Current Pattern
[当前留存模式]

### Likely Levers
- [杠杆]

### Risks or Friction
- [摩擦]

### Recommended Interventions
- [干预建议]
```

## Examples / 示例

好：

- “产品只在首次使用时提供价值，但没有形成任何 recurring trigger 把用户带回日常 workflow。”

差：

- “用户流失是因为大家都太忙了。”

## Common Pitfalls / 常见误区

- 把留存只当 CRM 问题
- 忽略协作或复用机制
- 在核心价值没修好前先做召回
