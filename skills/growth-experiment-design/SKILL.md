## Purpose / 用途

把增长实验设计成“一个改动、一个主指标、一个决策”。

## Key Concepts / 核心概念

好实验通常：

- 尽量隔离一个主变量
- 只盯一个 primary metric
- 事先写清决策规则

## Application / 用法

```markdown
## Growth Experiment

### Objective
[目标指标]

### Hypothesis
[假设]

### Change
[改动]

### Audience
[受众]

### Decision Rule
[继续 / 停止 / 迭代 的规则]
```

## Examples / 示例

好：

- “给新 team admin 加 checklist，只有当 setup completion 提升至少 10% 才继续扩量。”

差：

- “先试几个增长点子看看。”

## Common Pitfalls / 常见误区

- 一次改太多变量
- 成功标准太模糊
- 忽略样本量和时长
