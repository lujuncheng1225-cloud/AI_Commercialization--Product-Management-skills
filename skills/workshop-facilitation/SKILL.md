## Purpose / 用途

在正式产出之前，用结构化对话把问题、约束、假设和目标收束清楚。

## Key Concepts / 核心概念

### Entry Modes / 进入方式

- Guided：一轮一个问题
- Context dump：先接收一大段背景，再补关键缺口
- Best guess：在信息不足时基于假设快速推进

### Facilitation Rules / 引导规则

- 开始前先说明目标和方式
- 一轮只问一个高价值问题
- 问题尽量具体、口语化
- 在起草前先回显已知信息和假设

## Application / 用法

### Opening Pattern / 开场模式

1. 说明这次会话想产出什么
2. 说明会怎么问
3. 缺关键信息时，优先补核心上下文

### Progress Pattern / 进度模式

可以用简单进度标签：

- Context 1/4
- Decision 2/3
- Drafting

## Examples / 示例

好：

- “我会先问 4 个短问题，最后给一个主建议和一个备选方案。”

差：

- “请把所有背景、文档、目标、用户、路线图、历史都发给我。”

## Common Pitfalls / 常见误区

- 一开始就问太多问题
- 到最后才暴露关键假设
- 问题过于抽象
