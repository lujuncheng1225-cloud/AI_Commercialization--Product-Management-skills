# /write-prd

Generate a PRD that moves from context to requirements with clear scope.

## Workflow

1. Start with `workshop-facilitation` if context is ambiguous.
2. Use `problem-statement` to clarify the user and business problem.
3. Use `prd-development` to build the full document.
4. End with assumptions, risks, and next steps.

## Checkpoints

- Validate the problem before expanding solution detail.
- Keep at least one measurable success metric.
- Name out-of-scope items explicitly.
