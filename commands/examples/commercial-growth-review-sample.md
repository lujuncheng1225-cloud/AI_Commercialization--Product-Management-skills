# Sample Output: Commercial Growth Review

## 1. 关键问题定义

- 当前主要问题不是获客不足，而是 `trial -> activated -> paid` 这段转化效率偏低。
- 已有用户能走到核心价值，但在首次付费决策点流失较大。

## 2. 漏斗诊断

- `trial start -> activation` 转化尚可，说明入口价值表达不是主要瓶颈。
- `activation -> paid` 掉点最大，优先怀疑购买时机、套餐表达和升级路径摩擦。
- 如果用户在看到结果后仍未购买，优先检查 paywall 触发时机和价值锚定。

## 3. 留存评审

- 如果用户只完成一次关键任务后不回来，说明产品还没有形成重复使用理由。
- 需要区分“没买”是因为首单门槛高，还是因为根本没建立持续使用习惯。

## 4. 商业指标视图

- 主指标建议先看 `activation-to-paid conversion`。
- 配套看 `trial completion rate`、`first value time`、`7-day retained active users`。
- Guardrail 至少包含退款率、投诉率和关键流程完成率。

## 5. 实验建议

### P1

- 重做首次高意图时刻的升级触发，把付费理由和即时结果绑定。

### P2

- 测试更清晰的套餐对比页，弱化抽象权益，强化任务结果和使用边界。

### P3

- 对已激活未付费用户做一次轻量二次触达，验证回访是否能提升首单转化。

## 6. Decision

- `先做实验`

原因：

- 当前已经能定位主要问题区间，但还不能直接证明是价格、包装还是升级路径本身造成的，需要先用小实验缩小因果范围。

下一步 1-2 周动作：

- 确认漏斗基线数据
- 只选 1 个主指标启动 2 个实验
- 为每个实验定义成功阈值和停止规则
