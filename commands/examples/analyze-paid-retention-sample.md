# Sample Output: Analyze Paid Retention

## 1. 留存诊断

- 当前付费用户流失更可能发生在“首支作品完成后缺少下一次使用理由”。
- 如果产品价值主要发生在发歌前后，留存逻辑应围绕“持续发布”而不是日常高频打开。

## 2. 关键留存杠杆

- 让用户更容易为下一首歌复用同一工作流
- 为高质量用户保存模板、角色、风格和项目资产
- 提供持续创作的节奏型提醒，而不是泛 CRM 召回

## 3. VOC 线索

- 用户并不一定抱怨生成质量，而是抱怨“下次还要从头做”
- 付费用户更看重效率和稳定性，而不只是单次效果

## 4. 指标建议

- 主指标：paid user repeat creation rate
- 次指标：30-day paid retention、top-up repeat rate、第二次付费间隔
- Guardrails：项目完成率、负反馈率

## 5. 实验建议

- “复用上次项目风格”入口
- 为付费用户提供模板保存和快捷复用
- 针对已完成首支作品用户做下一首歌创作触达
