# Sample Output: Redesign Pricing Packages

## 1. 定价结构建议

- Free：低额度体验 + 基础模型 + 无并行
- Creator：适合单人创作者，含月度基础积分和高级模型入口
- Pro：高频创作者，含更高并行、更高质量导出、更优先队列
- Top-up Credits：给爆发式使用用户补充额度

## 2. 价值指标建议

- 优先考虑按“生成与导出能力”组合收费，而不是只卖抽象 credits。
- 如果保留积分，建议让用户清楚知道“一首作品通常消耗多少”。

## 3. 包装原则

- 套餐设计要围绕用户任务，而不是围绕模型列表。
- 高价值权益要清楚集中在一层，不要到处打散。

## 4. 风险

- 如果高阶权益表达不清，用户会继续只买小额补充包。
- 如果 Free 过强，会员价值不明显。
- 如果分层过于复杂，会伤害购买决策速度。

## 5. 落地建议

- 先测试 3 层方案，不要一次上太多档位。
- 结果页和额度不足时刻展示最匹配的套餐。
