# Sample Output: Redesign Pricing Packages

## 1. Value Metric 建议

- 主指标：按“生成与导出能力”组合收费，而不是只卖抽象 credits。
- 备选指标：按项目数或团队席位收费，适合协作型场景。
- 不建议：只按模型列表收费。用户买的是结果和交付能力，不是模型名称。

## 2. 定价结构建议

- Free：低额度体验 + 基础质量 + 单人使用
- Creator：适合轻量个人用户，含月度基础额度和标准导出
- Pro：适合高频用户，含更高额度、更快队列和高质量导出
- Team / Studio：适合多人协作或商用场景，含权限、协作和商用授权
- Top-up Credits：给短期爆发式使用用户补充额度

## 3. 免费层边界

- 免费层保留完整体验路径，但限制额度、导出质量或商用能力。
- 付费层承接高频使用、商用导出、团队协作和优先队列。
- 不要把最强结果能力完全放进免费层，否则付费理由会变弱。

## 4. 升级触发点

- 用量触顶：用户达到月度生成或导出上限。
- 质量触发：用户需要更高质量结果或更快处理。
- 商用触发：用户需要去水印、授权、团队协作或批量导出。

## 5. 包装原则

- 套餐设计要围绕用户任务，而不是围绕功能清单。
- 高价值权益要清楚集中在一层，不要到处打散。
- 每一档都要能回答：这档服务谁，为什么他会升级。

## 6. 风险

- 如果高阶权益表达不清，用户会继续只买小额补充包。
- 如果 Free 过强，会员价值不明显。
- 如果分层过于复杂，会伤害购买决策速度。

## 7. 落地建议

- 先测试 3 层方案，不要一次上太多档位。
- 结果页和额度不足时刻展示最匹配的套餐。
- rollout 前设置 guardrails：付费转化率、ARPPU、退款率和高价值用户降级率。
